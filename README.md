# Spam Detection Web Application

A beginner-friendly **MLOps demo project** built to demonstrate the full lifecycle of a machine learning application — from data preprocessing and model training to containerization, CI/CD automation, and cloud deployment.

> **University Project** — Students clone this repository, train the model locally, and deploy it to Docker, AWS, and Azure.

---

## 📌 Project Overview

This project builds a **text message spam classifier** using:

| Component | Technology |
|---|---|
| Machine Learning | Multinomial Naive Bayes |
| Feature Extraction | CountVectorizer (scikit-learn) |
| Web Framework | Flask (Python) |
| Frontend | HTML5 + CSS3 |
| Containerization | Docker |
| CI/CD | GitHub Actions |
| Cloud (Option A) | AWS EC2 |
| Cloud (Option B) | Azure Web Apps |

## 📊 Dataset Information

- **Name:** SMS Spam Collection Dataset
- **Source:** UCI Machine Learning Repository
- **Size:** 5,574 text messages (4,827 ham + 747 spam)
- **Format:** Tab-separated values (.tsv) with two columns:
  - `label` — `ham` (legitimate) or `spam`
  - `message` — The raw SMS text

**Download the dataset:**
```bash
# The dataset is automatically downloaded when you run the setup below.
# Or download manually from:
# https://archive.ics.uci.edu/ml/datasets/SMS+Spam+Collection
```

## 🛠️ Installation & Local Setup

### Prerequisites
- Python 3.9 or higher
- pip
- Git

### Step 1 — Clone the Repository
```bash
git clone https://github.com/YOUR_USERNAME/spam-detection.git
cd spam-detection
```

### Step 2 — Create a Virtual Environment
```bash

venv\Scripts\activate
```

```bash
pip install -r requirements.txt
```

Place the `SMSSpamCollection` file in the project root directory:

## ⚙️ GitHub Actions CI/CD

This project includes a GitHub Actions workflow (`.github/workflows/docker-build.yml`) that:
1. Triggers automatically on every push to the `main` branch
2. Builds the Docker image using the `Dockerfile`
3. Logs in to Docker Hub using your GitHub Secrets
4. Pushes the image to Docker Hub

### Setup GitHub Secrets

In your GitHub repository, go to:
**Settings → Secrets and variables → Actions → New repository secret**

Add these two secrets:

| Secret Name | Value |
|---|---|
| `DOCKER_USERNAME` | Your Docker Hub username |
| `DOCKER_PASSWORD` | Your Docker Hub password or access token |

Once set, every `git push` to `main` will automatically build and publish your Docker image. ✅

## ☁️ AWS Deployment (EC2)

Deploy your Docker container on an AWS EC2 Ubuntu instance.

### Step 1 — Launch an EC2 Instance
1. Go to [AWS Console](https://console.aws.amazon.com/) → **EC2** → **Launch Instance**
2. Choose **Ubuntu Server 22.04 LTS** (Free Tier eligible)
3. Choose **t2.micro** instance type
4. Under **Security Group**, add an inbound rule:
   - Type: **Custom TCP**, Port: **5000**, Source: **0.0.0.0/0**
5. Download your **`.pem` key file** (e.g., `my-key.pem`)
6. Launch the instance

### Step 2 — Connect via SSH
```bash
# Set correct permissions on your key file
chmod 400 my-key.pem

# Connect (replace with your EC2 Public IP)
ssh -i my-key.pem ubuntu@YOUR_EC2_PUBLIC_IP
```

### Step 3 — Install Docker on EC2
```bash
# Update packages
sudo apt-get update -y

# Install Docker
sudo apt-get install -y docker.io

# Start Docker service
sudo systemctl start docker
sudo systemctl enable docker

# Allow current user to run Docker without sudo
sudo usermod -aG docker ubuntu

# Log out and log back in for the group change to take effect
exit
```
Reconnect via SSH, then verify:
```bash
docker --version
```

### Step 4 — Pull Your Docker Image
```bash
# Pull the image from Docker Hub (replace YOUR_USERNAME)
docker pull YOUR_USERNAME/spam-detector:latest
```

### Step 5 — Run the Container
```bash
# Run the container in detached mode on port 5000
docker run -d -p 5000:5000 --name spam-app YOUR_USERNAME/spam-detector:latest
```

### Step 6 — Access the Application
Open your browser and go to:
```
http://YOUR_EC2_PUBLIC_IP:5000
```

> 💡 **Tip:** Make sure port 5000 is open in your EC2 Security Group inbound rules.

## 🔷 Azure Deployment (Web Apps)

Deploy your Docker container using Azure App Service — no SSH required!

### Step 1 — Go to Azure Portal
Go to [portal.azure.com](https://portal.azure.com) and sign in.

### Step 2 — Create a Web App
1. Click **+ Create a resource** → search for **Web App**
2. Click **Create**
3. Fill in the details:
   - **Subscription:** Your subscription
   - **Resource Group:** Create new → `spam-detection-rg`
   - **Name:** `spam-detector-app` *(must be globally unique)*
   - **Publish:** **Docker Container** ← important!
   - **Operating System:** Linux
   - **Region:** Choose closest to you
4. Click **Next: Docker →**

### Step 3 — Configure Docker
1. **Image Source:** Docker Hub
2. **Access Type:** Public
3. **Image and tag:** `YOUR_USERNAME/spam-detector:latest`
4. Click **Review + Create** → **Create**

### Step 4 — Deploy
Azure will automatically pull your image from Docker Hub and deploy it. This takes 1–3 minutes.

### Step 5 — Access the Application
Once deployment is complete, go to:
```
Overview → Default domain
```
Your app URL will look like:
```
https://spam-detector-app.azurewebsites.net
```

> 💡 **Tip:** Azure Web Apps for Linux containers listen on port **80** by default.  
> Set the `WEBSITES_PORT` environment variable to `5000` if needed:  
> **Configuration → Application settings → New setting:** `WEBSITES_PORT = 5000`
